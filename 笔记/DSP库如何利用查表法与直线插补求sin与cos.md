# DSP库如何利用查表法与直线插补求sin与cos

## 饱和运算

求-128-128的范围值的时候，当值超过128与-128时，那么就只求128与-128，称为饱和运算，在stm32中带有Q前缀就是饱和运算

## **查表法**

通过预先计算一系列离散的sin和cos值，并将它们存储在表中。这样，在需要计算sin或cos时，只需从表中读取相应的值，速度非常快。具体步骤如下：

1. **创建查找表**：
   - 预计算一组sin和cos值，并将这些值存储在数组中。例如，可以将角度范围从0到360度分成N个等分点，计算这些点上的sin和cos值，并将它们存储在表中。
2. **角度到表索引的映射**：
   - 将输入的角度转换为查找表中的索引。这通常是通过将角度转换为表中对应的离散点来完成的。
3. **查找并返回值**：
   - 从查找表中读取对应的sin或cos值。

## 直线插补

为了提高查表法的精度，直线插补可以用于在查找表的离散点之间进行插值。插值的目的是在已知点之间估算未知点的值，从而提高精度。具体步骤如下：

1. **创建查找表**：

   - 和查表法相同，首先创建一个包含预计算的sin和cos值的查找表。

2. **确定表索引和插值因子**：

   - 对于给定的角度，计算其在查找表中的下标位置。将角度映射到表中的两个最接近的点（即整数下标和下一个整数下标）。
   - 计算这两个点之间的插值因子（即角度与两个表点之间的距离）。

3. **插值计算**：

   - 使用插值因子对两个表值进行线性插值。具体公式如下：

### 代码解释

1. **输入范围转换**：

   ```
     in = x * 0.159154943092f;
   ```

   这行代码将输入角度 `x` 从弧度转换到一个0到1之间的比例，其中 `0.159154943092f` 是 `1 / (2 * PI)` 的近似值。这样做的目的是将输入值标准化为 `[0, 1]` 范围。因为输入的值是弧度值，弧度为角度*PI/180.0f,那么相乘1 / (2 * PI)就可以使得0-360°归一化

2. **计算整数部分 `n`**：

   ```
     n = (int32_t) in;
   ```

   `n` 是 `in` 的整数部分，表示输入角度在0到1之间的哪个区间。这对于处理周期性是必要的。

3. **处理负值**：

   ```
     if(x < 0.0f)
     {
       n--;
     }
   ```

   这段代码确保在处理负值时，`n` 会向下调整，以确保在查找表中能够正确地进行线性插值。

4. **映射到 `[0, 1]` 范围**：

   ```
     in = in - (float32_t) n;
   ```

   这里将 `in` 调整到 `[0, 1]` 范围，以便用于查找表的索引计算。

5. **计算查找表的索引**：

   ```
     findex = (float32_t) FAST_MATH_TABLE_SIZE * in;
     index = ((uint16_t)findex) & 0x1ff;
   ```

   FAST_MATH_TABLE_SIZE对应512，当前值求512可以求出对应值，然后采用过饱和运算求到index

   `findex` 计算了查找表的实际索引值，而 `index` 是在查找表中实际使用的索引。这里假设查找表的大小是 `FAST_MATH_TABLE_SIZE`，并且索引被限制在有效范围内（`0x1ff` 即511，假设查找表大小是512）。

6. **计算分数部分 `fract`**：

   ```
     fract = findex - (float32_t) index;
   ```

   cCopy Codefract 求出来浮点数减去整数，那么算出来就是小数部分

   `fract` 是在查找表中两个相邻点之间的相对位置，用于线性插值计算。

7. **线性插值**：

   ```
    a = sinTable_f32[index];
    b = sinTable_f32[index+1];
    sinVal = (1.0f-fract)*a + fract*b;
   ```

   从查找表中获取两个邻近点 `a` 和 `b` 的值，然后根据 `fract` 对这两个值进行线性插值，得到最终的正弦值。
