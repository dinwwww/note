## 调试过程

在调试过程有遇到第一个问题，半桥芯片有使能引脚，只有拉高，才会正常工作，因此有一个GPIO控制这个引脚，开发板不是GPIO直接IO控制这个引脚，如图

![image-20240927171406771](C:\Users\ding\AppData\Roaming\Typora\typora-user-images\image-20240927171406771.png)

但是这个引脚不是直接控制，是PF10控制GPIO再由GPIO控制到光耦隔离元件，才能导通拉高具体如图，而且这个电路是5V电压，5V电压不是由主控板子输出的，是由额外的外接电源降压而来，因此必须接到外接电源，霍尔元件和驱动电路才能正常工作。在这个电路中，光耦隔离，GPIO没有拉高，那么SD_IN就是默认高电平芯片IR停止输出，但是GPIO拉高后，SD_IN被地强行拉低到低电平，IR芯片正常工作。而且本电路有过流保护，在上下桥臂同时导通或者电流过大的时候，会触发这个过流保护，强行把光耦关断，关闭驱动芯片保护电路。

因此，在初始化的时候，不能直接打开PWM输出，与OC比较值，必须把下桥臂默认全部关闭，而且上桥臂PWM初始化，OC比较值也不能使PWM输出高电平，因为换向是在中断识别霍尔扇区，再开启与关闭PWM的，否则这样有可能导致触发过流保护，在触发中断前PWM设置由于是PWM模式1,所以进入中断是高电平，如果给了初值，那么在中断就是3路PWM都是高电平，然后在中断里面识别扇区，再开启下桥臂，这样就会导致某一相上下桥臂同时导通触发过流保护。

/**

 \* @brief    BLDC电机控制半桥芯片启动停止引脚初始化

 \* @param    无

 \* @retval    无

 */

 void bsp_BLDC_Init(void)

{

   GPIO_InitTypeDef GPIO_InitStructure;

  /* SHUTDOWN引脚时钟使能PF10 */

  TIMX_SHUT_DOWN_GPIO_CLK_ENABLE();

  /* 上桥臂GPIO配置 开启复用推挽输出 */

  GPIO_StructInit(&GPIO_InitStructure);

  GPIO_InitStructure.GPIO_Pin = TIMX_SHUT_DOWN_GPIO_PIN;

  GPIO_InitStructure.GPIO_Mode = GPIO_Mode_OUT;

  GPIO_InitStructure.GPIO_OType = GPIO_OType_PP;

  GPIO_InitStructure.GPIO_PuPd = GPIO_PuPd_NOPULL;

  GPIO_InitStructure.GPIO_Speed = GPIO_Low_Speed;

  GPIO_Init(TIMX_SHUT_DOWN_GPIO_PORT,&GPIO_InitStructure);

  GPIO_ResetBits(TIMX_SHUT_DOWN_GPIO_PORT,TIMX_SHUT_DOWN_GPIO_PIN);

}
